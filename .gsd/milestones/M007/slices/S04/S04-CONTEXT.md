## The Entry: Automatic Training Material Writer

### What was built

A new `docs/` module added to the sample tracking monorepo — a Next.js + Fumadocs documentation site that hosts end-user training materials. The initial content was generated by having agents read the full context of the platform:

- All Jira tickets and Confluence docs for sample tracking
- All Slack messages relating to the platform
- Full commit and PR history
- All E2E tests

That context was used to build a holistic understanding of what the platform is and how it works, then generate structured training guides (getting started, creating reports, working with reports, managing reports) published in the docs site.

A `training-material-writer` skill (`docs/content/docs/training-materials/`) was then created that runs on every PR open. It reads code changes plus related documentation (Jira tickets, PR description) and automatically:
- Generates new E2E tests for changed functionality
- Creates or updates training materials so users always have access to the latest documentation

### Where it's going

- **MCP knowledge base**: Training materials become documents in a knowledge base connected to the company's internal chat agent via MCP tools
- **In-app chat agent**: The agent sits inside the sample tracking platform so users can ask questions while using the app — e.g., "how do I do task B?" — and receive real tutorials generated from the knowledge base, with step-by-step navigation guidance
- **Guided walkthroughs**: The agent can eventually guide users through actual UI steps, not just describe them
- **AI agent training**: Training materials can instruct automated agents performing tasks on the platform
- **Screenshot-enriched docs**: E2E test screenshots for each workflow step can enrich training materials with real visuals
- **Product demos and launch materials**: Combining screenshots, E2E tasks, and docs enables automatic creation of product demos and feature launch materials on every code release

### Stack and tools

- Fumadocs (Next.js docs framework)
- Claude Code + GSD for the docs module build and skill creation
- Agent team mode for multi-source context gathering (Jira, Confluence, Slack, git history, E2E tests)
- `.agents/skills/training-material-writer/SKILL.md` — the PR-triggered skill
- `docs/FEATURES.md` — the knowledge base document generated from full platform context
