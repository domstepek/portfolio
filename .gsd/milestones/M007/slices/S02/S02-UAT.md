# S02: Engineering Journal Agent Skill — UAT

**Milestone:** M007
**Written:** 2026-03-14

## UAT Type

- UAT mode: artifact-driven
- Why this mode is sufficient: The slice deliverable is a static SKILL.md file (not a running service). Integration proof was achieved by writing a validation entry following the skill's exact output format and verifying it builds, parses, and renders through the S01 pipeline. No live runtime or human-experience verification is needed — correctness is mechanically verifiable.

## Preconditions

- `~/.agents/skills/engineering-journal/SKILL.md` exists
- `~/.gsd/agent/skills/engineering-journal` symlink exists and points to `~/.agents/skills/engineering-journal`
- Website repo is at `/Users/jstepek/Personal Repos/website/`
- S01 (enhanced markdown rendering and tag system) is complete

## Smoke Test

Run `test -f ~/.agents/skills/engineering-journal/SKILL.md && test -L ~/.gsd/agent/skills/engineering-journal && echo "PASS"` — confirms both the skill file and GSD symlink are present.

## Test Cases

### 1. Skill file has valid YAML frontmatter

1. Run `head -5 ~/.agents/skills/engineering-journal/SKILL.md`
2. **Expected:** Output contains `name: engineering-journal` and `description:` fields within `---` delimiters

### 2. GSD symlink resolves correctly

1. Run `readlink ~/.gsd/agent/skills/engineering-journal`
2. **Expected:** Output is `/Users/jstepek/.agents/skills/engineering-journal`

### 3. Skill appears in pi auto-discovery

1. Start a new pi session
2. Check `<available_skills>` in the system prompt
3. **Expected:** An entry with `name: engineering-journal` appears with the correct `location` path

### 4. Skill output format integrates with pipeline

1. Write a markdown file to `src/content/notes/test-journal.md` following the skill's frontmatter template (quoted title, summary, published date, tags array, type: journal)
2. Include a fenced code block with language tag
3. Run `pnpm build`
4. **Expected:** Build succeeds; route `/notes/test-journal` appears in output
5. Run `pnpm test`
6. **Expected:** All 18 Playwright tests pass
7. Remove `src/content/notes/test-journal.md`

### 5. Frontmatter parsing correctness

1. Write a journal entry with `tags: [webgpu, debugging]` and `type: journal`
2. Parse with gray-matter and apply `parseFrontmatter()` logic
3. **Expected:** `tags` is `["webgpu", "debugging"]` (array, not empty); `type` is `"journal"` (not defaulted to `"note"`)

## Edge Cases

### Title with special YAML characters

1. Write a journal entry with `title: "Debugging: why colons break YAML"`
2. Parse with gray-matter
3. **Expected:** Title preserved correctly including the colon

### Missing optional fields

1. Write a journal entry without `updated` field
2. Parse with gray-matter and apply `parseFrontmatter()` logic
3. **Expected:** Entry builds successfully; `updated` is undefined/null (not an error)

## Failure Signals

- SKILL.md missing or symlink broken → pi won't discover the skill
- YAML frontmatter missing `name:` → skill may not appear in agent discovery
- Skill output generates tags as a string instead of array → notes render with empty tag list (silent failure)
- Skill output omits `type: journal` → note defaults to `type: 'note'` on the index page (silent failure)
- Build failure when adding a journal entry → frontmatter schema mismatch or markdown pipeline error

## Requirements Proved By This UAT

- R501 — Skill file exists and is discoverable; generates entries from conversation context (proven by file inspection and output format validation)
- R506 — Skill includes media convention for `public/notes/<slug>/` and evidence TODO markers (proven by SKILL.md content inspection)

## Not Proven By This UAT

- Tone quality of generated entries — the skill instructs agents on tone, but actual quality depends on the executing agent's adherence. This is inherent to prompt-based tools and not mechanically verifiable.
- Real conversation-to-journal workflow — the validation entry was hand-written following the skill format, not generated by an agent executing the skill from actual session context. A full end-to-end test would require invoking the skill in a real coding session.

## Notes for Tester

- Test case 4 creates a temporary file — make sure to clean it up after testing.
- `parseFrontmatter()` silently defaults malformed fields. If tags render as empty or type shows as "note" for a journal entry, the frontmatter is likely malformed — check YAML syntax carefully.
- The skill is a static prompt file. There's no "running" it in isolation — it's consumed by agent tools (pi, Cursor, etc.) when the user invokes it.
